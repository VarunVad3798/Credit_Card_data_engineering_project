# OpenBank Credit Card Platform - Synthetic Data

This is a fictional, synthetic dataset designed for data engineering practice.

## Business Scenario

A digital bank processes credit card applications through the OpenBank Platform. Applications are evaluated by a FICO-based decision engine. Approved applications create card accounts. Card transactions and account payments are generated by downstream operational systems.

The data engineering workstream must integrate data from source platforms into an AWS data lake and an Enterprise Data Warehouse.

## Source Systems

- OPENBANK_PLATFORM
- FICO_DECISION_ENGINE
- CARD_PLATFORM
- CARD_PROCESSOR
- PAYMENTS_PLATFORM

## Files

| File | Purpose |
|---|---|
| customer_master.csv | Customer master data |
| credit_card_applications.csv | Credit card application events |
| fico_credit_decisions.csv | FICO and credit decision results |
| credit_card_accounts.csv | Approved card accounts |
| card_transactions.csv | Card transaction activity |
| account_payments.csv | Customer payment activity |
| data_mapping.csv | Source-to-target mapping example |

## Intentional Data Quality Issues

The dataset includes a small number of deliberate issues:

- FICO score outside the valid 300-850 range
- Missing application ID in FICO data
- Duplicate transaction ID
- Transaction referencing a non-existent account
- Negative payment amount
- Invalid customer email format

These records are intended for rejected-record and data-quality pipeline testing.

## Suggested Data Quality Rules

- application_id must not be null
- customer_id must exist in customer master
- fico_score must be between 300 and 850
- approved_credit_limit must be non-negative
- transaction_id must be unique
- transaction account_id must exist in card accounts
- payment_amount must be greater than zero
- customer email should match a valid email pattern
